Archipelago¶

class pagmo::archipelago¶

Archipelago.

An archipelago is a collection of pagmo::island objects which provides a convenient way to perform multiple optimisations in parallel.

Public Types

using size_type = size_type_implementation¶

The size type of the archipelago.

This is an unsigned integer type used to represent the number of islands in the archipelago.

Public Functions

archipelago()¶

Default constructor.

The default constructor will initialise an empty archipelago.

archipelago(const archipelago &other)¶

Copy constructor.

The islands of other will be copied into this via archipelago::push_back().

Parameters
  • other: the archipelago that will be copied.
Exceptions
  • unspecified: any exception thrown by archipelago::push_back().

archipelago(archipelago &&other)¶

Move constructor.

The move constructor will wait for any ongoing evolution in other to finish and it will then transfer the state of other into this. After the move, other is left in an unspecified but valid state.

Parameters
  • other: the archipelago that will be moved.

template <typename... Args, n_ctor_enabler< Args... > = 0>
archipelago(size_type n, Args&&... args)¶

Constructor from n islands.

NOTE: this constructor is enabled only if the parameter pack Args can be used to construct a pagmo::island.

This constructor will forward n times the input arguments args to the push_back() method.

Parameters
  • n: the desired number of islands.
  • args: the arguments that will be used for the construction of each island.
Exceptions
  • unspecified: any exception thrown by the invoked pagmo::island constructor or by archipelago::push_back().

~archipelago()¶

Destructor.

The destructor will call archipelago::wait() internally, and run checks in debug mode.

island &operator[](size_type i)¶

Mutable island access.

This subscript operator can be used to access the i-th island of the archipelago (that is, the i-th island that was inserted via push_back()). References returned by this method are valid even after a push_back() invocation. Assignment and destruction of the archipelago will invalidate island references obtained via this method.

NOTE: the mutable version of the subscript operator exists solely to allow calling non-const methods on the islands. Assigning an island via a reference obtained through this operator will result in undefined behaviour.

Return
a reference to the i-th island of the archipelago.
Parameters
  • i: the index of the island to be accessed.
Exceptions
  • std::out_of_range: if i is not less than the size of the archipelago.

size_type size() const¶

Size.

Return
the number of islands in the archipelago.

template <typename... Args, push_back_enabler< Args... > = 0>
void push_back(Args&&... args)¶

Add island.

NOTE: this method is enabled only if the parameter pack Args can be used to construct a pagmo::island.

This method will construct an island from the supplied arguments and add it to the archipelago. Islands are added at the end of the archipelago (that is, the new island will have an index equal to the value of size() before the call to this method).

Parameters
  • args: the arguments that will be used for the construction of the island.
Exceptions
  • unspecified: any exception thrown by memory allocation errors or by the invoked constructor of pagmo::island.

void evolve(unsigned n = 1)¶

Evolve archipelago.

This method will call island::evolve() on all the islands of the archipelago. The input parameter n represent the number of times the run_evolve() method of the island’s UDI is called within the evolution task.

Parameters
  • n: the parameter that will be passed to island::evolve().
Exceptions
  • unspecified: any exception thrown by island::evolve().

void wait()¶

Block until all evolutions have finished.

This method will call island::wait() on all the islands of the archipelago.

void get()¶

Block until all evolutions have finished and raise the first exception that was encountered.

This method will call island::get() on all the islands of the archipelago. If an invocation of island::get() raises an exception, then on the remaining islands island::wait() will be called instead, and the raised exception will be re-raised by this method.

Exceptions
  • unspecified: any exception thrown by any evolution task queued in the archipelago’s islands.

bool busy() const¶

Check archipelago status.

Return
true if at least one island is evolving, false otherwise.
